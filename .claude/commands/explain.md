# Explain Command

## Purpose

Explain project structure, code, or technical decisions in a clear and understandable way.

The goal is to help developers understand the system before making changes.


## Explanation Approach

Before explaining:

1. Identify the requested topic.
2. Inspect relevant files.
3. Understand dependencies and relationships.
4. Use existing project documentation.


## Explanation Levels

Adapt the explanation depending on the request.

Possible levels:

### Overview

Explain:

- purpose
- responsibilities
- main components
- relationships


### Developer Level

Explain:

- implementation details
- important classes
- data flow
- extension points


### Architecture Level

Explain:

- module boundaries
- dependencies
- design decisions
- trade-offs


## Rules

When explaining:

Prefer:

- simple language
- practical examples
- diagrams when useful
- references to existing files


Avoid:

- unnecessary theory
- generic explanations
- assumptions without checking code


## Output Format

Provide:

## Purpose

What this component does.

## Location

Where it exists in the project.

## How It Works

Step-by-step explanation.

## Important Decisions

Why it was designed this way.

## Possible Improvements

Only suggest improvements that provide real value.

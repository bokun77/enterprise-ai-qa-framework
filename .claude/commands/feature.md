# Feature Command

## Purpose

Create a new feature following AI QA Framework development principles.

The goal is to implement changes safely and incrementally.

---

## Workflow

Follow these steps:

### 1. Understand Request

Analyze:

- What problem are we solving?
- What behaviour is expected?
- Which modules are affected?

Do not start coding immediately.

---

### 2. Analyze Existing Code

Before implementation:

- Inspect related modules.
- Find existing patterns.
- Identify reusable components.

Avoid creating duplicate solutions.

---

### 3. Architecture Check

Verify:

- Correct module ownership.
- Dependency direction.
- Impact on existing design.

If a major architectural change is required, explain it first.

---

### 4. Implementation

Implement:

- Minimal required changes.
- Existing coding conventions.
- Appropriate tests.

Prefer small focused commits.

---

### 5. Verification

Run:

- Compilation
- Existing tests
- New tests

Report:

- What changed
- What was tested
- Possible risks

---

## Output Format

Provide:

## Feature Summary

What was implemented.

## Design Decision

Why this approach was chosen.

## Files Changed

List modified files.

## Verification

Tests and checks performed.

## Next Steps

Possible improvements if needed.

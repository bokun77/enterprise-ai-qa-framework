# Review Command

## Purpose

Perform a structured review of code changes.

The goal is to maintain quality and prevent unnecessary complexity.

---

## Review Process

Analyze:

### Architecture

Check:

- Module placement
- Dependencies
- Design consistency
- Complexity

---

### Code Quality

Check:

- Naming
- Readability
- Maintainability
- Duplication

---

### Testing

Check:

- Test coverage
- Test stability
- Assertion quality
- Framework usage

---

### AI Principles

Check:

- Were existing components reused?
- Were unnecessary files created?
- Was complexity introduced without reason?

---

## Output Format

Provide:

## Summary

Short overview.

## Findings

Problems and recommendations.

## Risks

Potential future issues.

## Decision

Choose:

- Approved
- Approved with suggestions
- Needs changes
